# Phase 3 — Point Field Visual Language

## Purpose

Choose how the point field looks at rest and how it reacts under the already-approved field model. This phase determines the resting grid language and the visible character of the point response.

## Dependencies

- Phase 2 motion language is approved
- Field behavior remains fixed during this phase
- Any visual changes are limited to mapping and point treatment

## Objective

Approve a point/grid treatment that:
- feels intentional at rest
- stays calm behind foreground content
- preserves the field-first aesthetic rather than turning into a graphic effect

## Scope

- Evaluate point appearance and grid character
- Choose the rest-state language
- Decide whether points remain perfect circles or evolve slightly

## Phase deliverables

- Approved point style
- Approved grid density and spacing model
- Approved primary / secondary mapping mix

## Exit criteria

- Background feels intentional at rest
- Ripple response remains visible but non-dominant
- Point treatment feels premium rather than gimmicky

## Stage breakdown

### Stage 3.1 — Rest-state grid exploration

**Goal**
Define the visual language of the field before interaction occurs.

**Implementation tasks**
- Compare a strict Cartesian grid, a softened regular grid, and a slightly staggered grid
- Keep tonal treatment and point count comparable during evaluation
- Assess which rest-state feels most premium behind content

**Checks**
- The field does not feel empty or accidental at rest
- The chosen grid supports later deformation cleanly
- The pattern does not feel overly rigid or overly decorative

### Stage 3.2 — Mapping exploration

**Goal**
Choose the cleanest mapping mix for the approved field.

**Implementation tasks**
- Compare displacement only
- Compare displacement plus size
- Compare displacement plus size plus restrained brightness
- Keep all versions calibrated against the same field profile

**Checks**
- The primary response still reads as spatial displacement
- Secondary channels support the effect without stealing focus
- One mapping mix clearly emerges as the calmest strong option

### Stage 3.3 — Point appearance exploration

**Goal**
Resolve the visual identity of the point itself.

**Implementation tasks**
- Compare perfect circular points
- Compare softened-edge circles
- Compare very slight point deformation only if visually justified
- Judge each option under both rest state and repeated interaction

**Checks**
- Point treatment feels refined, not gimmicky
- Shape changes do not introduce visual noise
- The chosen point language remains stable at multiple sizes and densities

### Stage 3.4 — Density and spacing tuning

**Goal**
Find the calmest field density that still reads as continuous.

**Implementation tasks**
- Tune point count and spacing together
- Avoid moire-like visual noise under motion
- Validate behavior on both large and small viewports
- Define default density rules and any responsive adjustments

**Checks**
- Density is high enough to feel field-like
- Density is low enough to stay calm and legible
- No viewport size creates accidental clutter or emptiness

## Hand-off to next phase

At the end of Phase 3, the point field should have an approved rest-state language. Phase 4 can then add controlled irregularity without re-opening:
- the motion profile
- the grid density baseline
- the primary mapping hierarchy
